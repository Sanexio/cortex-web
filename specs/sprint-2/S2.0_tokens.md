## §1 Auftrag (in 1 Satz)

Alle `--pxz-*` Design-Tokens in **eine** zentrale Datei `assets/css/tokens.css`
konsolidieren, bestehende Duplikate in `style.css` und `homepage.css` entfernen,
`karriere.css` auf Referenzen umstellen — **ohne eine einzige sichtbare
Veränderung** auf Homepage oder Karriere-Seite.

## §2 Phase 1 — Verständnis-Sicherung

### §2.1 IST-Stand (harte Zahlen, gemessen 2026-04-18)

| Datei | Token-Deklarationen | Scope | Zeilen |
|---|---|---|---|
| `style.css` | 9 Primitives (`--pxz-red/red-soft/red-dark/ink/graphite/mist/line/chalk/snow`) | `:root` | Z.28–38 |

| **Verbrauch** | **171 `var(--pxz-*)`-Zugriffe** | 4 Dateien | — |

### §2.2 Probleme (FK-5 Kontextverlust + PXZ-E-001-Risiko)

1. **Split-Location**: `--pxz-red`, `--pxz-ink` etc. in `style.css` UND `homepage.css` parallel. Änderung an einer Stelle driftet still.
2. **Karriere isoliert**: `karriere.css` verwendet Hex direkt (`#F5B800`), statt Referenz auf `--pxz-amber`. Sprint-3-Mehrsprachigkeit oder Theme-Wechsel zukünftig teurer.
3. **Keine globalen Tokens**: Eine Standard-Textseite (z.B. kommendes Datenschutz-Template ohne eigenes CSS) hat aktuell keine `--pxz-bg`, `--pxz-text` etc. Sprint 2.3 Batch A würde sie neu erfinden.

### §2.3 Zielzustand

- **Genau eine** `tokens.css` auf `:root` (Primitives + semantische Light-Defaults).
- `style.css`, `homepage.css` enthalten **keine Primitives mehr**.
- `homepage.css` behält nur den Light-Scope auf `.pxz-home` (Font-Stack, Box-Sizing) und den Dark-Override auf `.pxz-mfa`. Semantische Defaults kommen via Vererbung von `:root`.
- `karriere.css` referenziert Primitives mit `var()`. Karriere-spezifische semantische Werte (`--pxz-bg: #0E0E10`, Textschleier) bleiben im `.pxz-kar`-Scope.
- **171 `var(...)`-Zugriffe** liefern weiterhin exakt dieselben Werte.

### §2.4 Constraints (hart)

- **0 px optisches Delta** — verify.sh §3+§4 bleibt grün, ab-diff.mjs Delta ≤ Jitter.
- Token-Namen ändern sich NICHT (sonst 171 Edits).
- Keine Änderung an Card-Klassen (`.pxz-mfa-card`, `.pxz-loc-card--main`, `.pxz-kar-card`) — explizit späterer Sprint-2-Task.
- Keine neue WP-Option, kein DB-Schema-Change, keine Plugin-Änderung.
- PXZ_VERSION bumpt 2.7.4 → 2.7.5 (patch, reine Architektur).

### §2.5 Implizite Annahmen (explizit gemacht)

1. `:root` Custom Properties vererben durch den gesamten DOM-Baum, auch in Sektions-Scopes wie `.pxz-home` und `.pxz-mfa`. Overrides auf `.pxz-mfa { --pxz-bg: … }` funktionieren weiter, weil Custom Properties vererben.
2. Tokens.css muss in Cascade VOR style.css + homepage.css + karriere.css landen. Sicherung via WP-Enqueue-Dependency-Kette.
3. Der Karriere-Schwarzton `#0E0E10` ist **designgetrieben** verschieden von `--pxz-ink` (#1D1D1F) — bleibt Hex in `.pxz-kar`-Scope. Bei Bedarf später eigenes Primitive `--pxz-ink-deep` (out of scope S2.0).

## §3 Phase 2 — Lösungsdesign

### §3.1 Neue Datei `assets/css/tokens.css`

```css
/* =============================================================================
 * Praxiszentrum Design Tokens — Single Source of Truth
 * Version via PXZ_VERSION (functions.php) + ?ver= cache-buster.
 * References: DESIGN_GUIDELINES §5 (Farbsystem), §13 (Spacing-System).
 *
 * Layer 1: Primitives  — neutrale Werte (Farben), keine Semantik
 * Layer 2: Semantic    — Light-Mode-Defaults; Sektions-/Seiten-Scopes
 *                        (.pxz-home, .pxz-mfa, .pxz-kar) überschreiben diese
 *                        bei Bedarf (Dark-Mode, Akzent-Wechsel).
 * ============================================================================= */

:root {
  /* --- Layer 1: Primitives ------------------------------------------------ */
  --pxz-white:        #FFFFFF;
  --pxz-snow:         #FBFBFD;
  --pxz-chalk:        #F5F5F7;
  --pxz-line:         #D2D2D7;
  --pxz-mist:         #86868B;
  --pxz-graphite:     #424245;
  --pxz-ink:          #1D1D1F;
  --pxz-red:          #C8161D;
  --pxz-red-soft:     #E8484F;
  --pxz-red-dark:     #8B0F14;
  --pxz-amber:        #F5B800;
  --pxz-amber-bright: #FFD700;

  /* --- Layer 2: Semantic Defaults (Light-Mode) ---------------------------- */
  --pxz-bg:           var(--pxz-white);
  --pxz-text:         var(--pxz-ink);
  --pxz-text-muted:   var(--pxz-graphite);
  --pxz-text-tertiary:var(--pxz-mist);
  --pxz-accent:       var(--pxz-red);
  --pxz-accent-hover: var(--pxz-red-dark);
}
```

### §3.2 Änderung `functions.php` (Cascade-Ordnung absichern)

```php
add_action( 'wp_enqueue_scripts', function() {
    // Design tokens FIRST — must load before any consumer stylesheet.
    wp_enqueue_style( 'pxz-tokens', PXZ_URI . '/assets/css/tokens.css', [], PXZ_VERSION );

    wp_enqueue_style( 'blocksy-parent', get_template_directory_uri() . '/style.css', [ 'pxz-tokens' ], PXZ_VERSION );
    wp_enqueue_style( 'praxiszentrum',  get_stylesheet_uri(),                         [ 'blocksy-parent' ], PXZ_VERSION );

    if ( is_page_template( 'template-homepage.php' ) ) {
        wp_enqueue_style( 'pxz-homepage', PXZ_URI . '/assets/css/homepage.css', [ 'praxiszentrum' ], PXZ_VERSION );
    }
    if ( is_page_template( 'template-karriere.php' ) ) {
        wp_enqueue_style( 'pxz-karriere', PXZ_URI . '/assets/css/karriere.css', [ 'praxiszentrum' ], PXZ_VERSION );
    }
}, 20 );
```

**Warum Dependency-Kette:** `pxz-tokens` → `blocksy-parent` → `praxiszentrum` → `pxz-homepage`/`pxz-karriere`. WordPress serialisiert Enqueues gemäß Deps. Damit ist garantiert, dass `tokens.css` vor allen anderen in die Cascade eingespielt wird.

Zeilen 27–38 löschen (Kommentar + `:root`-Block). Rest der Datei (Utilities, Blocksy-Reset, Lang-Switcher) bleibt.

Die 17 Token-Deklarationen entfallen, alle verbleibenden `var(--pxz-*)`-Zugriffe in der Datei funktionieren über die geerbten `:root`-Tokens. Der `.pxz-mfa { --pxz-bg: var(--pxz-ink) … }`-Override (Z.237ff) bleibt **unverändert** — Custom Properties vererben in den Subtree.

### §3.5 Änderung `assets/css/karriere.css`

Zeilen 20–37 (`.pxz-kar { --pxz-* … }`) ersetzen durch:

```css
.pxz-kar {
  /* Karriere is a pauschal dark page. Semantic tokens overridden
   * for the whole .pxz-kar subtree; primitives come from :root (tokens.css). */
  --pxz-accent:        var(--pxz-amber);
  --pxz-accent-hover:  var(--pxz-amber-bright);
  --pxz-bg:            #0E0E10;                /* deeper than --pxz-ink by design (amber glow pairing) */
  --pxz-text:          rgba(255,255,255,0.92);
  --pxz-text-muted:    rgba(255,255,255,0.7);
  --pxz-text-tertiary: rgba(255,255,255,0.5);
  background: var(--pxz-bg);
  color: var(--pxz-text);
  font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text", "Inter", "Helvetica Neue", Helvetica, Arial, sans-serif;
  font-size: 17px;
  line-height: 1.5;
  min-height: 100vh;
}
```

Die 4 Primitive-Deklarationen (`--pxz-white`, `--pxz-ink`, `--pxz-amber`, `--pxz-amber-bright`) entfallen — sie kommen aus `:root`. Der feste Karriere-Hintergrund `#0E0E10` bleibt Hex mit Kommentar-Begründung.
