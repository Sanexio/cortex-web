## §1 Verständnis

### Aktueller Zustand (IST)

- Blocksy-Parent rendert einen minimalen `<footer id="footer" class="ct-footer">`
  mit nur einem `data-column="copyright"`-Block.
- Child-Theme `praxiszentrum` hat **kein** `footer.php` → Blocksy rendert.
- `register_nav_menus(['footer' => 'Footer-Navigation'])` ist bereits in
  `functions.php` registriert, aber kein WP-Menü zugewiesen.
- Footer ist damit funktional leer (kein Impressum-Link, keine Kontaktdaten,
  kein Sprechzeit-Hinweis).

### Probleme

1. Rechtssicherheit: Impressum + Datenschutz sollten aus Footer erreichbar sein.
2. UX: Nutzer erwartet Adressblock, Sprechzeiten, Telefon im Footer.
3. Brand-Konsistenz: Header ist vollständig PXZ-gebrandet, Footer ist Blocksy-Default.

### Nicht-Scope

- Blog-Footer-Widgets (Blocksy hat welche, wir brauchen keine)
- Footer-Search
- Social-Media-Icons (Praxis hat keine aktiven Social-Kanäle — später)

## §2 Lösungsdesign

### F-1 Override-Strategie

**Entscheidung:** Child-Theme-`footer.php` überschreibt Blocksy-Parent.
Das ist WP-Standard-Mechanismus (`get_footer()` sucht zuerst im aktiven Theme).

**Alternativen:**
- (a) Blocksy-Customizer-Footer belassen + Widgets füllen → abgelehnt,
  da Customizer-Config nicht git-trackbar.
- (b) `footer.php` im Child → **gewählt**, git-trackbar, versioniert, tokens-only.

### F-2 Struktur

4 Spalten auf Desktop (≥1100px), 2 Spalten auf Tablet (690-1099px), 1 Spalte auf Mobile.
Darunter eine Bottom-Bar (immer horizontal).

```
│  Grüneburgweg  │  Tel allg.     │  Mo·Di·Do      │  - Praxis     │

```

### F-3 Datenquellen

- **Adresse/Telefon/E-Mail**: `inc/practice-data.php` (SSoT, S2.3-kern)
- **Navigation**: `inc/nav-data.php` (SSoT, S2.4), gleiche Struktur wie Header
- **Rechts-Nav + Subfooter-Texte**: **NEU** `inc/footer-data.php` (4 Sprachen)

### F-4 Dateien

| Datei | Typ | Zweck |
|---|---|---|
| `footer.php` | NEU | Override Blocksy-Parent |
| `template-parts/site-footer.php` | NEU | Footer-Markup (aufgerufen aus footer.php) |
| `inc/footer-data.php` | NEU | Rechts-Nav + Subfooter-Texte |
| `assets/css/footer.css` | NEU | Styles, tokens-only |
| `functions.php` | MOD | require_once footer-data + enqueue footer.css global |

### F-5 HWG-Konformität

- Keine Preise
- Kein „bester Arzt / beste Praxis"
- Keine Heilversprechen
- Adresse, Tel, Öffnungszeiten, Navigation = neutrale Sachangaben → unbedenklich

### F-6 Brand-Switch

`/team/` + alle 8 Arzt-Slugs zeigen „Praxisgemeinschaft Sanexio" im Header-Title.
Footer soll **einheitlich die Praxis-Wortmarke** beibehalten — der Footer
gehört zur Praxis-Site, nicht zur Sanexio-Marke. Nur das Copyright-Jahr ist
dynamisch (`date('Y')`).

## §3 Umsetzung

### T-1 `inc/footer-data.php` (NEU)
Rechts-Nav + Subfooter-Texte, 4 Sprachen, Struktur analog `nav-data.php`.

### T-2 `template-parts/site-footer.php` (NEU)
Rendert das 4-Spalten-Grid + Bottom-Bar.

### T-3 `footer.php` (NEU)
Ruft `get_template_part('template-parts/site-footer')` auf, schließt `wp_footer()`.

### T-4 `assets/css/footer.css` (NEU)
- Dark-Theme (Praxis-Palette-Teal-Dark als Hintergrund)
- 4-Spalten-Grid mit `minmax(0, 1fr)`
- Mobile-Stack mit Media-Query
- Tokens-only (keine Hex/RGB-Literale außer wo notwendig für Kontrast)

### T-5 `functions.php` (MOD)
- `require_once PXZ_PATH . '/inc/footer-data.php';` nach nav-data
- `wp_enqueue_style('pxz-footer', ...)` global, nach nav.css
- PXZ_VERSION bump 2.7.19 → 2.7.20

## §4 Selbstprüfung (Akzeptanzkriterien)

| # | AK | Prüfmethode |
|:-:|---|---|
| AK-1 | footer.php überschreibt Blocksy | `curl /praxis/` → kein `class="ct-footer"` mehr, stattdessen `class="pxz-footer"` |
| AK-2 | 4 Spalten auf Desktop sichtbar | HTML hat 4× `.pxz-footer-col` |
| AK-3 | Adresse+Tel+E-Mail klickbar | `tel:+49…`, `mailto:praxis@…`, Doctolib-Button-Link |
| AK-4 | Sprechzeiten aus practice-data | Werte aus `pxz_practice_data_for_lang()` |
| AK-5 | Navigation aus nav-data | Alle 7 primary-Items ohne children, flach |
| AK-6 | Impressum + Datenschutz erreichbar | 2× `<a>` in `.pxz-footer-bottom` |
| AK-7 | Copyright-Jahr dynamisch | HTML enthält `© 2026` (current year) |
| AK-8 | Mobile-Stack | Viewport 375px: alle Spalten untereinander, Bottom-Bar 2-zeilig |
| AK-9 | smoke-seo 21/21 bleibt grün | `tools/smoke-seo.sh` |
| AK-10 | smoke-http 5/5 bleibt grün | `tools/smoke-http.sh` |
| AK-11 | Home + Karriere unverändert | MD5-Match auf homepage + karriere-archive (oder Delta nur durch neue footer-Section) |
| AK-12 | HWG-konform | 0 Preise, 0 „bester", 0 Heilversprechen in Footer-HTML |
