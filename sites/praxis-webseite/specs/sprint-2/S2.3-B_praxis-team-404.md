## §4 SEO-Strategie übergreifend

| Technik | Umsetzung |
|---|---|
| **Eindeutiger H1** | Pro Seite genau 1 `<h1>`, kein `<h1>` in Hero-Subline |
| **Semantische Tags** | `<section>`, `<article>`, `<nav>` (nav nur im Header/Footer), `<aside>` nur wo inhaltlich gerechtfertigt |
| **Meta-Tags** | Über `wp_head`-Hook, NICHT Yoast/AIOSEO-Plugin (Local-WP hat AIOSEO — wir liefern unsere Tags als Override, Plugin-Felder lassen wir leer) |
| **Schema.org** | JSON-LD im `<head>`, MedicalClinic / AboutPage, keine Microdata-Inlines |
| **Alt-Texte** | `template-standard.php` hat aktuell keine Bilder (Hero ist text-only), keine Alts nötig in Batch B |
| **Canonical** | `<link rel="canonical">` per `wp_head`-Filter |
| **Open Graph** | `og:title`, `og:description`, `og:type`, `og:url`, `og:site_name` |
| **Interne Verlinkung** | Praxis → Team, Aerzte, Sprechstunden, Karriere; Team → Aerzte, Karriere; 404 → Home, Sprechstunden, Kontakt |
| **URL-Slugs** | `/praxis/` und `/team/` — kurz, sprechend, deutsch (LL-041 Tutorial-Hierarchie-Leitprinzip auch für URLs) |
| **Lesbarkeit** | Absätze ≤ 4 Sätze, Listen statt Wand-Absätze, Sub-Headings alle ~150–200 Wörter |

### §4.1 SEO-Helper (neue Datei `inc/seo-meta.php` im Theme)

Damit Meta/OG/Schema wiederverwendbar pro Template-Typ setzbar ist:

```php
// Pseudocode — genaue Umsetzung in Phase 3
function pxz_seo_meta() {
  if ( is_page('praxis') )    pxz_render_meta( $praxis_seo_data );
  if ( is_page('team') )      pxz_render_meta( $team_seo_data );
  if ( is_404() )             pxz_render_meta( $s404_seo_data );
}
add_action('wp_head', 'pxz_seo_meta', 5);
```

SEO-Daten als PHP-Array in `inc/seo-data.php`, pro Seite ein Eintrag. Klar getrennt vom Template-Code → testbar.
