## §2 Lösungsdesign

### F-1 Card-Hover normalisieren

Alle vier Card-Patterns auf dasselbe Rezept:

```css
transition: transform 180ms cubic-bezier(0.2, 0, 0, 1),
            box-shadow 180ms cubic-bezier(0.2, 0, 0, 1);

:hover,
:focus-visible {
  transform: translateY(-3px);
  box-shadow: var(--pxz-shadow-card-hi);
}
```

Warum 180ms cubic-bezier(0.2, 0, 0, 1)?
- 180ms fühlt sich bei Card-Interaktionen nicht träge an, aber wahrnehmbar
- cubic-bezier(0.2, 0, 0, 1) ≈ Material's "Standard Easing", schnell loslegen,
  weich auslaufen — lebendiger als das default `ease`
- -3px statt -2px: mehr spürbarer Lift, bei gleichzeitigem Shadow noch subtil

### F-2 Drawer-Easing iOS-artig

```css
.pxz-nav-drawer          { transition: transform 0.32s cubic-bezier(0.32, 0.72, 0, 1); }
.pxz-nav-drawer-backdrop { transition: opacity   0.32s cubic-bezier(0.32, 0.72, 0, 1); }
```

### F-3 Diagnostik-Card-Typo

```css
.pxz-diagnostik-hub-card-title {
  font-size: clamp(1.25rem, 2vw, 1.5rem);
  line-height: 1.18;
  letter-spacing: -0.01em;
  font-weight: 600;
}
```

Ergibt 20px Mobile → 24px Desktop, harmonisch mit `.pxz-title-3`-Kaskade
(die 22px-30px hat), aber eine Stufe kleiner weil Card-Titel.
