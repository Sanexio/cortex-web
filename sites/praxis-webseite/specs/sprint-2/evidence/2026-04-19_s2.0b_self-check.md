## Karriere-Delta (dokumentierte Abweichung von C1)

Karriere zeigt −9 px Dimensionsdelta auf allen 3 Viewports, lokalisiert auf den
`.wpforms-submit`-Button. Ursache: generisches `.pxz-btn` aus `components.css`
greift jetzt global (vorher nur auf Home über `homepage.css`). Werte-Delta:

| Property | v2.7.7 | v2.7.8 | Bewertung |
|---|---|---|---|
| `height` | 58 px | 49 px | konsequent mit Design-System |
| `line-height` | 25.5 px (`1.5`-Faktor) | 17 px (`1.0`-Faktor) | CSS-Konvention für Buttons |
| `min-height` | 40 px | 44 px | **WCAG 2.5.5 Target Size** |

**Architekten-Entscheidung:** akzeptiert als Accessibility-Gewinn, nicht
als Regression. Vollständige Analyse in `md5-diff.md`.

## In-Session entdecktes Problem + Lösung (S2.0b-LL-1)

Beim ersten Post-Refactor-MD5-Vergleich zeigten alle 6 Home/Karriere-Shots
Dimensionsdelta. Root-Cause-Analyse über Puppeteer-`getComputedStyle` lokalisierte
das Problem auf `.pxz-eyebrow` und `.pxz-sect-intro`, deren margins durch die
Reset-Regel `.pxz-home :where(p) { margin: 0 }` in `homepage.css` überschrieben
wurden, weil components.css jetzt vor homepage.css in der Kaskade liegt und die
Regeln identische Spezifität (0,1,0) haben.

**Fix:** Page-scoped Overrides in `homepage.css` direkt nach der Reset-Regel:
```css
.pxz-home .pxz-eyebrow { margin-bottom: 1.25rem; }
.pxz-home .pxz-sect-intro { margin-top: 1.75rem; }
```
Spezifität 0,2,0 > Reset 0,1,0 → greift zuverlässig unabhängig von
Stylesheet-Ladereihenfolge. Nach diesem Fix: Home-MD5 3/3 MATCH.

## Lessons Learned

- **S2.0b-LL-1** — Spezifitäts-neutrale `:where()`-Resets schützen nur innerhalb
  derselben Datei vor class-basierten margin-Overrides. Wird eine class-Regel in
  eine früher geladene Kaskaden-Datei (Schicht 3) verschoben, muss entweder der
  Reset vollständig `:where()`-gewrappt werden (mit Nebenwirkungen für Elemente
  ohne eigenen Override) oder ein page-scope-Override mit erhöhter Spezifität
  ergänzt werden (empfohlen).

- **S2.0b-LL-2** — Puppeteer-Full-Page-Screenshots sind nicht immer
  deterministisch; `tablet768`-Viewport zeigte bei identischem CSS gelegentlich
  Pixel-Jitter (SSIM 0.988736 gemessen). Für MD5-Null-Delta-Beweise **zusätzlich**
  Dimension-Delta + Computed-Style-Probe + ggf. Sub-Element-`offsetHeight`
  messen — das ermöglicht Trennung von Rausch und echter Regression.

- **S2.0b-LL-3** — Beim Heraufziehen von Komponenten auf globale Schicht 3 wird
  jede page-spezifische Kaskade-Annahme ("meine homepage.css-Regel kommt nach
  dem Reset, also gewinnt sie") zur architektonischen Schuld. Empfehlung: bei
  jeder Klassen-Promotion die Kaskaden-Interaktion mit nicht-promotierten
  Resets explizit prüfen (grep auf `:where(p)`, generische Tag-Selektoren,
  `* { margin: 0 }`-Resets in allen Layer-übergreifenden Dateien).

- **S2.0b-LL-4** — Globale Komponenten-Regeln (`.pxz-btn`) bringen
  Accessibility-Properties wie `min-height: 44px` auf **alle** Seiten, die
  vorher nur auf einzelnen Seiten griffen. Das ist meistens ein Gewinn, kann
  aber Dimensions-Delta auf vorher-unbetroffenen Pages erzeugen. In der Spec
  sollte C1 (Null visuelle Änderung) für **jede** Seite separat spezifiziert
  werden, nicht pauschal.

- **S2.0b-LL-5** — WP-CLI funktioniert nicht out-of-the-box mit Local-by-
  Flywheel wegen des sandboxed MySQL-Sockets. Direkter `mysql`-Client-Aufruf
  über den LBF-internen Binärpfad (`…/lightning-services/mysql-…/bin/darwin-
  arm64/bin/mysql`) + Socket-Datei (`…/Local/run/<site-id>/mysql/mysqld.sock`)
  funktioniert. Pattern empfiehlt sich für künftige Draft-Page-Anlagen.
